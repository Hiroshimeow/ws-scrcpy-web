How should modal open/close animate?

Native <dialog> removes from top layer on .close() — closing animation needs special handling

A

Open animation only

Keep the current fade-in + slide-in on open. Close is instant — no exit animation. Simple, zero complexity. This is what most native dialog implementations do.

How: CSS dialog[open] selector triggers keyframe animation. Close calls .close() directly.
Lines of code: ~10 CSS (reuse existing keyframes)
B

Both open and close animations

Fade + slide in on open, reverse on close. Requires delaying .close() until the CSS animation finishes — the base class plays the exit animation, waits for animationend, then actually closes.

How: Add .closing class → play reverse animation → listen for animationend → call .close()
Lines of code: ~25 CSS + ~10 JS in base class
Gotcha: Must handle fast open-close (user clicks X immediately). Must handle confirm dialogs that cancel the close.
C

CSS @starting-style + transitions (modern API)

Uses the new @starting-style rule + transition-behavior: allow-discrete for both open and close. Pure CSS, no JS timing. But only works in Chromium 117+ and Firefox 129+.

How: Pure CSS — @starting-style defines entry state, transitions handle both directions automatically
Lines of code: ~20 CSS, 0 JS
Gotcha: No Safari support yet (coming in Safari 18.4+). But this app already requires WebCodecs, which has similar browser requirements.
